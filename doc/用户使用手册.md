# 运维环境轻量化自动发布系统 用户使用手册

## 1. 适用对象

本文档面向日常使用本系统进行 Jar 发布的运维、测试和研发人员。

---

## 2. 打开系统

在浏览器访问部署好的 Server 地址，例如：

```text
http://127.0.0.1:8080
```

进入页面后，你会看到三个主要区域：

- Agent 状态
- Jar 管理
- 发布操作
- 发布历史

---

## 3. 页面说明

### 3.1 Agent 状态

页面顶部会显示当前 Agent 连接状态：

- `Agent 在线`：可以执行发布
- `Agent 离线`：不能执行发布

如果在线，还会显示当前可管理的设备列表。

### 3.2 Jar 管理

用于上传可发布的 Jar 文件。

需要填写：

- `服务名`
- `Jar 文件`

上传成功后，页面会提示：

```text
上传成功：<service_name> / <filename>
```

并在“可用 Jar 列表”中新增一条记录。

### 3.3 发布操作

用于选择服务、Jar 和目标设备，然后触发发布。

需要填写：

- `服务名`
- `目标 Jar`
- `目标设备`

点击“开始发布”后：

- 按钮会变成 `发布中...`
- 任务面板会显示任务 ID 和当前状态

### 3.4 发布历史

用于查看历史发布记录，包括：

- 时间
- 服务名
- Jar 名
- 目标设备
- 总体状态
- 失败详情

---

## 4. 正常发布流程

### 步骤 1：上传 Jar

1. 在“Jar 管理”区域输入服务名
2. 选择一个非空 `.jar` 文件
3. 点击“上传 Jar”

注意：

- 当前系统只允许上传 `.jar` 后缀文件
- 空文件会被直接拒绝

### 步骤 2：选择发布参数

1. 在“发布操作”区域输入同样的服务名
2. 在下拉框中选择已上传的 Jar
3. 勾选目标设备

### 步骤 3：开始发布

1. 点击“开始发布”
2. 观察任务状态从 `执行中` 变为终态

可能的终态有：

- `成功`
- `失败`
- `部分失败`

### 步骤 4：查看结果

结果可以从两个位置看：

- 当前任务状态面板
- 发布历史列表

---

## 5. 如何判断发布是否成功

### 页面判定

如果任务状态是：

- `成功`：表示所有目标设备都成功
- `部分失败`：表示部分设备成功、部分设备失败
- `失败`：表示全部失败，或任务在 Agent/下载阶段就失败

### 设备结果判定

任务面板中会显示每台设备的结果，例如：

```text
device-b: success
device-c: failed (启动超时)
```

### 历史记录判定

在“发布历史”里可以看到：

- 总体状态
- 每台设备的状态
- 失败原因

---

## 6. 常见失败场景

### 6.1 Agent 离线

表现：

```text
发布失败：Agent 离线
```

处理方式：

- 确认 Agent 进程是否运行
- 确认 Server 和 Agent 网络连通

### 6.2 上传空文件

表现：

```text
上传失败：jar file is empty
```

处理方式：

- 重新选择一个非空 Jar

### 6.3 服务启动超时

表现：

```text
device-b: failed (启动超时)
```

处理方式：

- 检查远端启动脚本是否可执行
- 检查 `process_name` 是否配置正确

### 6.4 服务停止超时

表现：

```text
device-b: failed (停止超时)
```

处理方式：

- 检查远端停服脚本是否真正结束了旧进程

### 6.5 Jar 与服务不匹配

表现：

- 触发发布时报错
- 发布任务不会创建成功

处理方式：

- 确认发布时填写的 `服务名` 与 Jar 上传时使用的服务名一致

---

## 7. 推荐使用方式

建议每次发布按下面顺序操作：

1. 先确认页面顶部显示 `Agent 在线`
2. 上传目标 Jar
3. 确认“可用 Jar 列表”里能看到新文件
4. 选择正确服务名和目标设备
5. 点击“开始发布”
6. 等待任务进入终态
7. 在“发布历史”中复核结果

---

## 8. 典型示例

假设你要发布 `ops-demo-service` 到 `device-b`：

1. 在 Jar 管理中填写：
   - 服务名：`ops-demo-service`
   - 文件：选择你的 `.jar`
2. 点击“上传 Jar”
3. 在发布操作中填写：
   - 服务名：`ops-demo-service`
   - 目标 Jar：选择刚上传的文件
   - 目标设备：勾选 `device-b`
4. 点击“开始发布”
5. 等待显示：

```text
状态：成功
device-b: success
```

---

## 9. 注意事项

- 只有 Agent 在线时才能正常发布
- 一个服务同一时间只能有一个执行中的任务
- 上传 Jar 时，服务名必须和后续发布时保持一致
- 不要上传空文件
- 发布失败时优先查看页面上的设备级错误信息

---

## 10. 反馈与排查建议

如果页面出现异常，建议按以下顺序排查：

1. 刷新页面，确认 Agent 状态
2. 查看任务面板中的失败原因
3. 查看“发布历史”里的最近一条记录
4. 联系维护人员进一步查看 Agent 日志与远端设备状态
